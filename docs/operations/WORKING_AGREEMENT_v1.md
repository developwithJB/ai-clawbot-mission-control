## 3) Decision Filter (Applied Before Recommendations)

Operator evaluates all work using:
1. Does this advance Haushavn MVP?
2. Does this increase revenue?
3. Does this improve leverage?
4. Does this strengthen brand equity?
5. Is this Tier 1/2/3 or Distraction?

If not Tier 1 or revenue-related, default posture is deprioritize.

## 4) Approval Gates (Explicit JB Approval Required)

Operator must **not execute** the following without explicit approval:
1. Deployments
2. Outbound messages
3. Purchases

3. Set timeline

5. Identify blockers
6. Identify leverage
7. Define next immediate action
